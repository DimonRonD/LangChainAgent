# LangChainAgent

Простой локальный агент на базе LangChain + Ollama, который принимает вопросы на естественном языке и решает арифметические задачи через инструменты.

## Что умеет проект

- Работает локально через `ChatOllama`.
- Поддерживает инструмент `add` (сложение).
- Поддерживает инструмент `subtract` (вычитание).
- Поддерживает инструмент `multiply` (умножение).
- Поддерживает инструмент `divide` (деление, с защитой от деления на ноль).
- Поддерживает инструмент `power` (возведение в степень).
- Поддерживает инструмент `sqrt` (квадратный корень, с проверкой отрицательных значений).
- Запускается в интерактивном CLI-режиме.
- Пишет лог запроса к модели, сырого ответа и финального результата.

## Структура

- `main.py` — основная логика агента, инструменты, интерактивный цикл, логирование.
- `requirements.txt` — Python-зависимости проекта.
- `.env.example` — пример переменных окружения.
- `.env` — локальные значения переменных окружения (не коммитится).

## Переменные окружения

Пример (`.env`):

```env
OLLAMA_MODEL=llama3.1
OLLAMA_TEMPERATURE=0
OLLAMA_BASE_URL=http://localhost:11434
LOG_LEVEL=INFO
```

## Установка и запуск

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
python main.py
```

Для выхода из агента: `exit` или `quit`.

Примеры запросов:

- `сколько будет 2 плюс 3`
- `10 минус 4`
- `7 умножить на 8`
- `22 разделить на 11`
- `5 в степени 3`
- `корень из 81`

## Ответы на вопросы по архитектуре

### 1) Какая цель у агента?

Агент — это локальный CLI-помощник для арифметики на естественном языке. Пользователь задает вопрос текстом, агент определяет, нужен ли инструмент, и возвращает результат.

### 2) Какие компоненты LangChain используются?

- **LLM:** `ChatOllama` — языковая модель, работающая через Ollama.
- **Инструменты:** `@tool`-функции `add`, `subtract`, `multiply`, `divide`, `power`, `sqrt`.
- **Сообщения:** `HumanMessage` для передачи пользовательского ввода.
- **Агент-экзекьютор:** создается через `build_agent(...)`, где используется:
  - `langchain.agents.create_agent`, либо
  - fallback: `langgraph.prebuilt.create_react_agent`.
- **Промпт:** явный кастомный system prompt в коде не задан, используется стандартное поведение выбранного агентного конструктора плюс текст пользователя.

### 3) Где происходит «принятие решений» агентом?

Основная точка принятия решений — вызов `agent.invoke(payload)` в `ask_agent()`. Внутри этого шага агент/модель решает:

- ответить сразу текстом;
- или выбрать инструмент (`add`/`subtract`/`multiply`/`divide`/`power`/`sqrt`) и передать параметры.

Дополнительно есть fallback-слой: если модель вернула tool-call как JSON-строку, функция `_run_tool_call_from_text(...)` вручную распознает этот вызов и исполняет соответствующий инструмент.
